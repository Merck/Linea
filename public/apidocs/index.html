    Introduction

    Welcome to the Linea API! This API follows standards of JSON API.

    API Basics

    Resource object

    # object
  "data": {
    "id": "286",
    "type": "datasets",
    "links": {
      "self": "http://localhost:3000/api/datasets/286"
    },
    ...

# object attributes
    "attributes": {
      "name": "mytest3 - DELETE ME",
      "description": "mytest2",
      "created-at": "2016-02-04T12:28:19.206Z"
    },

# relationships
    "relationships": {
      "terms-of-service": {
        "links": {
          "self": "http://localhost:3000/api/datasets/286/relationships/terms-of-service",
          "related": "http://localhost:3000/api/datasets/286/terms-of-service"
        }
      },

# links
        "links": {
          "self": "http://localhost:3000/api/datasets/286/relationships/subject-area",
          "related": "http://localhost:3000/api/datasets/286/subject-area"
        }
      },

    “Resource objects” appear in a JSON API document to represent resources.

    A resource object MUST contain at least the following top-level members:

    • id
    • type

    Exception: The id member is not required when the resource object originates at the client and represents a new resource to be created on the server.

    In addition, a resource object MAY contain any of these top-level members:

    • attributes – an attributes object representing some of the resource’s data.
    • relationships – a relationships object describing relationships between the resource and other JSON API resources.
    • links – a links object containing links related to the resource.
    • meta – a meta object containing non-standard meta-information about a resource that can not be represented as an attribute or relationship.

    API

    You must always include correct headers:

    • Content-Type: application/vnd.api+json
    • Accept: application/vnd.api+json; version=1

    when calling our API.

    API versioning

    Our API is versioned. The difference - version - is set by sending version=1 keyword in Accept: header.

    Accept: application/vdn.api+json; version=1

    Authentication

    curl -XPOST "api/sessions" \
  -H "Content-Type: application/vnd.api+json" \
  -H "Accept: application/vdn.api+json; version=1" \
  -d '{"username":"_username_","password":"_password"}'

{"api_key":"valid API key goes here"}

    POST /api/sessions

    Every user must be authenticated prior using our API. Authentication is against IAM. If you need access to the API, send an e-mail to Linea admins.

    Parameter Required Description
    username true username of authenticated user
    password true password of authenticated user

    Authorization

    curl -X GET "api_endpoint_here" \
  -H "Content-Type: application/vnd.api+json" \
  -H "Accept: application/vdn.api+json; version=1" \
  -H "Authorization: valid API key goes here"

    Make sure to replace valid API key goes here with your given API key from authentication.

    Linea API expects for the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: valid API key goes here

    API key is valid for 15 minutes. Every single request prolongs this timeout.
    You must replace valid API key goes here with your API key.

    HTTP Errors

    Error Code Meaning
    400 Bad Request – Your request sucks
    401 Unauthorized – Your API key is wrong
    403 Forbidden – You have no access there
    404 Not Found – The specified resource is not found
    405 Method Not Allowed – You tried to access resource with an invalid method
    406 Not Acceptable – You requested a format that isn’t json
    410 Gone – The resource requested has been removed from our servers
    418 I’m a teapot
    429 Too Many Requests – You’re requesting too many resources! Slow down!
    500 Internal Server Error – We had a problem with our server. Try again later.
    503 Service Unavailable – We’re temporarially offline for maintanance. Please try again later.

    dataset-attributes extended attributes
    ?include=dataset-attributes
    notes dataset notes
    ?include=notes

    ?include=tags

    Accessing relationships

    Note links part of json response. It contains several related URLs. So, to fetch relationships, you can use these URLS.

    • GET api/datasets/550/tags – will fetch all tags for dataset ID 550
    • GET api/datasets/550/notes – will fetch all notes for dataset ID 550
      • GET api/datasets/550/notes?include=user – will fetch notes including note owner

    You can use includes as well.

    Includes

    curl -X GET "/api/datasets/8?includes=owner" \
  -H "Content-Type: application/vnd.api+json" \
  -H "Accept: application/vdn.api+json; version=1" \
  -H "Authorization: valid API key goes here"

    {
  "data": {
    "id": "550",
    "type": "datasets",
    "links": {
      "self": "http://localhost:3000/api/datasets/550"
    },
    ...
    "relationships": {
      "owner": {
        "links": {
          "self": "http://localhost:3000/api/datasets/550/relationships/owner",
          "related": "http://localhost:3000/api/datasets/550/owner"
        },
        "data": {
          "type": "users",
          "id": "38"
        }
      },
      ...
    }
  },
  "included": [
    {
      "id": "38",
      "type": "users",
      "links": {
        "self": "http://localhost:3000/api/users/38"
      },
      "attributes": {
        "first-name": "First",
        "last-name": "Last",
        "email": "first.last@Linea.com"
      }
    }
  ]
}

    By default, there is no detailed listing of relations. If you need to fetch complete dataset record, use parameter include=

    • dataset-attributes - extended attributes
    • dataset-attributes.user - extended attributes owner
    • owner - owner of the dataset (User resource)
    • notes - dataset notes
    • notes.user - dataset notes including note creator
    • tables - tables
    • tables.columns - Tables resource including columns

    So, HTTP request is then GET /api/datasets/:id?include=notes,owner,tables.columns

    Example of included owner

    GET datasets/8?include=owner
